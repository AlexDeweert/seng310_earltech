//A requirement is a statement about an intended product that specifies what it //should do or how it should perform //A user requirement is what the project should be able to do. The software will do the following for the specific stakeholder/user Student: 1) Log in to coursespaces account on mobile or desktop and navigate to CalendarCompanion for access, creating a unique CalendarCompanion account for the user. 2) Allow student to configure desired degree map/schedule through multiple ways. -The student may chat and provide details (faculty, preferred Graduating Year, number of courses per semester) to the Virtual Assistant to automatically compile a chronological sequence of semesters based on user preferences. -Whether a user is stuck navigating through the application or is having trouble setting up a semester, the Virtual Assistant will be able to resolve any difficulties encountered. -The student has the option to modify or fine tune the semesters in order to obtain the perfect schedule for each studying term. This can be done through toggling data fields while designing their degree map. Additionally, courses booked online or through exchange will also be recognized by the CalendarCompanion -The software then displays each semester the user has designed intuitively through multiple data representations (graphs, pie charts, interfaces, map of UVIC). 3)By saving the desired degree schedule/map, CalendarCompanion will notify the student and remind them before course registration begins, reminding them to sign up for their respective planned term courses. 4)For international students, the CalendarCompanion provides multiple languages. 5)If a course is full, the CalendarCompanion may recalibrate your degree map in order to graduate as efficiently as possible 6)If a course is deemed terminated for a specific degree requirement, notify the students to update their schedule/degree map 7)The software is able to determine if degree requirements are not fulfilled within a degree map/schedule. Students will be notified to make immediate changes. 8)If a student accidentelly deletes a course by accident, there is an undo button to revert to the previous degree map 9)If a student deletes a mandotary pre/co-requisite to another course, a border will apply to said course with a message deeming that they must complete a specific pre/co-requisite before enrolling in said class. Administration (UVIC staff, faculty, professors): 1)Log in to coursespaces account on mobile or desktop and navigate to CalendarCompanion for access, creating an administrative CalendarCompanion account. 2)Update new courses and lecture sections for students to select from throughout the years. Fill in all data fields relevant to the courses updated so that the users (students) may have optimal course selection. 3)Delete future courses/sections if deemed that the course is dropped from the university itself. Parents 1)Log into the CourseCalendar using V number of specified child and monitor course map and transcripts
